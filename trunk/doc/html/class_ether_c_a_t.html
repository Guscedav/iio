Detailed Description

This class implements a simple EtherCAT master stack to communicate with devices on an EtherCAT fieldbus.
This master stack may use an Ethernet driver to transmit and receive EtherCAT frames. Using an Ethernet driver is required for hard-realtime communication.
Optionally, this master stack may use the TCP/IP stack of the operating system to transmit EtherCAT datagrams with multicast UDP packets. In order to receive responses from EtherCAT slave devices, the host system must be able to receive its own UDP packets. On a linux system, this must be enabled as follows: 


echo 1 > /proc/sys/net/ipv4/conf/eth0/accept_local

In this example, 'eth0' is the name of the Ethernet interface to use.
The desired Ethernet communication channel is chosen by using the appropriate constructor of this class. The first constructor requires a reference to an Ethernet driver, and the second constructor accepts the IP address of the interface to use for UDP communication.

◆ sendDatagrams()

void EtherCAT::sendDatagrams ( std::vector< Datagram * >  datagrams)

Assembles an EtherCAT frame from the list of given datagrams and transmits that EtherCAT frame on the fielbus. Calling this method also reads back the received (processed) datagrams, meaning that the given datagrams will be modified by the EtherCAT slave devices after calling this method.

Parameters
datagramsa list of datagrams to transmit on the EtherCAT fieldbus.
